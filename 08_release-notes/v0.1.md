## Release Purpose

This release establishes the first public version of the Agent Execution Boundary framework layer.

The purpose of this repository is to define why autonomous agent execution does not create source authority, authorship, provenance, approval, or valid action by itself.

This release names the control layer above autonomous execution.

It establishes the distinction between what an agent can do and what an agent has authority to do.

## Core Claim

Execution capacity is not source authority.

An agent may execute inside a frame.

The agent does not become the source of the frame.

Autonomy is not authority.

Automation is not authorship.

Recursive operation is not provenance.

Output is not approval.

Output is not provenance.

Use does not transfer Origin.

Valid action requires authority.

## Primary Distinction

This release separates execution from source authority.

The agent may act.

The action may be useful.

The output may be fluent.

The workflow may complete.

The loop may improve.

The implementation may function.

None of those create source authority.

Execution remains below the source-defined frame.

## Control Layer

This release defines the control layer as the layer above agent execution.

The control layer defines:

- authority
- scope
- approval
- evidence
- refusal
- escalation
- stop conditions
- provenance
- accountability

The control layer must exist before delegated agent execution becomes valid.

## Agent Action on Behalf of a User

This release does not prohibit agents from acting on behalf of a user.

It defines the conditions under which agent action on behalf of a user becomes valid.

An agent may act for the user when authority has been explicitly delegated within a defined scope.

The agent may not become the user, replace the user’s authority, approve its own action as if the user approved it, or redefine the source boundary.
