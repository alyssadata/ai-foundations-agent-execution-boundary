# AI Foundations | Agent Execution Boundary
# Stop Condition Requirement

## Definition

A stop condition is the defined point at which an agent must pause, halt, refuse, escalate, or return control to the proper authority.

A stop condition prevents delegated execution from becoming uncontrolled continuation.

An agent may execute inside a defined frame.

The agent must stop when the boundary requires it.

## Core Claim

Every delegated agent loop requires a stop condition.

An agent may not continue indefinitely because it is capable of continuing.

An agent may not expand scope because the next step appears useful.

An agent may not treat silence as permission to continue.

An agent may not treat workflow momentum as authority.

Delegated execution must include a defined stopping point.

## Why This Boundary Exists

Autonomous agents can chain actions.

They can continue from one task to the next.

They can infer adjacent steps.

They can optimize toward completion.

They can keep going when the original instruction did not authorize continuation.

Without a stop condition, the agent may drift from delegated action into unauthorized control.

The stop condition protects the boundary between valid execution and overreach.

## What a Stop Condition Does

A stop condition defines when the agent must stop acting.

It may require the agent to:

- pause
- hold
- refuse
- ask for approval
- request clarification
- escalate to authority
- return a draft
- log uncertainty
- preserve the current state
- avoid reality-changing action

Stopping is not failure.

Stopping can be the valid action.

## Stop Condition Before Execution

A stop condition must be defined before execution begins.

The system must know what stops the agent before the agent starts acting.

Without a stop condition, the agent may continue based on momentum, inference, confidence, tool access, or perceived usefulness.

None of those create authority.

A valid agent loop requires a defined end state.

## Required Stop Triggers

An agent must stop when:

- approval is missing
- authority is unclear
- scope is unclear
- evidence is insufficient
- sources conflict
- instructions conflict
- protected material appears
- provenance may be erased
- source-position may be blurred
- private mechanics may be exposed
- action may change reality without approval
- the task exceeds delegated limits
- the agent cannot determine validity

These triggers protect the control layer.

## Approval Stop

The agent must stop when approval is required but has not been granted.

A draft may be prepared.

A recommendation may be made.

A plan may be offered.

A summary may be returned.

The agent may not convert proposed action into approved action.

Output is not approval.

If approval is missing, the stop condition applies.

## Authority Stop

The agent must stop when authority is unclear.

The agent may not infer authority from:

- access
- capability
- prior patterns
- confidence
- workflow availability
- silence
- urgency
- usefulness
- generated reasoning

Authority must be defined.

If authority is unclear, the stop condition applies.

## Scope Stop

The agent must stop when the task exceeds delegated scope.

The agent may not expand the task because an adjacent action seems helpful.

The agent may not change related material without authority.

The agent may not continue into a new category of action without approval.

The agent may not assume that because one action was approved, all related actions are approved.

Scope expansion requires approval.

If scope is exceeded, the stop condition applies.

## Evidence Stop

The agent must stop when evidence is insufficient.

The agent may not act on:

- stale information
- weak information
- missing information
- assumed facts
- unclear instructions
- unsupported claims
- unverifiable sources
- incomplete context
- contradictory records

If the evidence does not support the action, the stop condition applies.

## Provenance Stop

The agent must stop when provenance may be erased, blurred, generalized, or replaced.

The agent may not remove attribution.

The agent may not generalize source-position.

The agent may not convert authored work into generic framework language.

The agent may not treat generated output as provenance.

The agent may not replace citation with summary.

Output is not provenance.

If provenance is at risk, the stop condition applies.

## Source-Position Stop

The agent must stop when source-position may be altered, obscured, generalized, or transferred.

The agent may not rewrite Origin into a generic role.

The agent may not imply that source-position is transferable.

The agent may not treat implementation as origin.

The agent may not treat adoption as authorship.

The agent may not treat agent-generated summary as source.

Use does not transfer Origin.

If source-position is at risk, the stop condition applies.

## Protected Material Stop

The agent must stop when protected material may be exposed.

Protected material may include:

- private mechanics
- protected calibration
- internal continuity tests
- private source architecture
- unreleased protocols
- living Continuum-contact
- private emotional architecture
- non-public approval logic
- internal drift repair structures

This repository is public-safe.

The protected core is not disclosed.

If protected material appears, the stop condition applies.

## Reality-Changing Stop

The agent must stop before reality-changing action unless authority has approved the action or pre-authorized that action class.

Reality-changing action may include:

- sending
- publishing
- deleting
- modifying
- assigning
- approving
- rejecting
- updating records
- changing public language
- triggering external workflows
- altering attribution
- altering source representation

If reality-changing action is not approved, the stop condition applies.

## Conflict Stop

The agent must stop when instructions conflict.

Conflict may occur between:

- user instruction and source boundary
- agent goal and approval requirement
- workflow speed and evidence requirement
- public release and private protection
- tool access and permission
- generated output and provenance
- delegated task and source-position
- automation and authorship

When conflict appears, the agent must not resolve the conflict by acting beyond authority.

The agent must stop or escalate.

## Drift Stop

The agent must stop when recursive operation begins to drift from the source-defined frame.

Drift may appear when the agent:

- simplifies away source language
- removes attribution
- generalizes specific authority
- treats boundaries as optional
- optimizes for fluency over provenance
- rewrites protected language without approval
- expands scope through inference
- continues after task completion

Recursive operation is not provenance.

If drift appears, the stop condition applies.

## Valid Stop Behavior

Valid stop behavior may include:

- returning a draft
- asking for approval
- requesting clarification
- flagging insufficient evidence
- identifying a boundary conflict
- preserving current state
- refusing unauthorized action
- escalating to human/source authority
- logging why action did not proceed

A controlled agent does not only know how to act.

A controlled agent knows when not to act.

## Invalid Continuation

Invalid continuation occurs when an agent continues after the stop condition has been reached.

Invalid continuation may include:

- acting without approval
- continuing beyond scope
- using available tools without permission
- treating silence as consent
- treating confidence as authority
- publishing from a draft-only state
- modifying records without authorization
- removing attribution
- rewriting protected language
- optimizing away source-position

Invalid continuation can appear helpful.

Helpfulness does not make invalid continuation valid.

## Stop Condition and User Delegation

This framework does not prohibit agents from acting on behalf of a user.

It defines the conditions under which agent action on behalf of a user becomes valid.

An agent may act for the user when authority has been explicitly delegated within a defined scope.

The agent must stop when delegated authority is missing, unclear, exceeded, or in conflict with the source boundary.

The agent may not become the user, replace the user’s authority, approve its own action as if the user approved it, or redefine the source boundary.

## Stop Condition and Source Authority

Source authority remains above delegated execution.

The stop condition protects the source-defined frame.

The agent may execute inside the frame.

The agent may not continue beyond the frame.

The agent may prepare action.

The agent may not approve its own action.

The agent may produce output.

The output does not become authority.

## Operational Test

Ask:

What makes the agent stop?

If the stop condition is defined before execution and triggers when authority, scope, evidence, approval, provenance, or source-position is at risk, the boundary may be valid.

If the agent can continue without a defined stop condition, the boundary has failed.

## Secondary Test Questions

Ask:

- What is the agent’s end state?
- What stops the agent?
- What requires approval?
- What requires escalation?
- What happens when evidence is insufficient?
- What happens when scope is unclear?
- What happens when provenance is at risk?
- What happens when source-position is at risk?
- What happens when protected material appears?
- Can the agent continue without approval?

## Pass Condition

The boundary passes when the agent stops at the defined boundary.

The agent does not continue by inference.

The agent does not treat access as permission.

The agent does not treat silence as consent.

The agent does not treat confidence as authority.

The agent does not continue beyond delegated scope.

The agent stops, holds, refuses, asks, or escalates when the stop condition is triggered.

## Failure Condition

The boundary fails when the agent continues after the stop condition should have applied.

Failure occurs when:

- approval is missing and the agent acts
- authority is unclear and the agent proceeds
- scope is exceeded and the agent continues
- evidence is insufficient and the agent executes
- provenance is at risk and the agent rewrites
- protected material appears and the agent exposes it
- source-position is blurred and the agent generalizes it
- reality-changing action occurs without valid authority
