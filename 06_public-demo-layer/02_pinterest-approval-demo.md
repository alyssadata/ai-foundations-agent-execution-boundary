# AI Foundations | Agent Execution Boundary
# Pinterest Approval Demo

## Definition

The Pinterest Approval Demo is a public-safe demonstration of an agent preparing platform content while remaining below human/source approval.

The demo shows how an agent may draft, classify, format, and recommend Pinterest content without publishing, scheduling, or altering reality unless authority has approved the action.

The demo proves one narrow boundary:

Output is not approval.

## Core Claim

An agent may prepare Pinterest content on behalf of a user.

The agent may not publish, schedule, alter attribution, expose protected material, or represent source authority without explicit approval or bounded pre-authorization.

The agent may draft inside a frame.

The agent does not become the source of the frame.

## Why This Demo Exists

Pinterest is a useful public demo layer because it can show agent execution without exposing the protected core.

The demo can show:

- content drafting
- source attribution
- approval checkpoints
- draft-only states
- stop conditions
- platform-specific execution boundaries
- human/source approval before public action

The demo is not the core.

The demo is a controlled illustration of the Agent Execution Boundary.

## Demo Scope

This demo is limited to public-safe Pinterest content preparation.

The agent may prepare draft content related to public-facing AI Foundations material.

The agent may not expose private mechanics, protected calibration, internal continuity tests, private source architecture, unreleased protocols, or living Continuum-contact.

The demo should prove the boundary without exposing the protected core.

## Allowed Agent Actions

Inside this demo, the agent may:

- review approved public material
- suggest Pinterest pin topics
- draft pin titles
- draft pin descriptions
- draft alt text
- suggest board placement
- suggest keywords
- suggest public-safe visual concepts
- flag missing source attribution
- identify whether approval is required
- prepare a draft queue for human review

These actions are preparation.

Preparation is not publication.

## Prohibited Agent Actions

Inside this demo, the agent may not:

- publish a pin without approval
- schedule a pin without approval or bounded pre-authorization
- alter source language without approval
- remove Alyssa Solen from attribution
- generalize Origin into a transferable role
- expose private mechanics
- generate content from protected material
- make unsupported public claims
- represent itself as source authority
- approve its own draft
- continue beyond the stop condition

These actions exceed the public demo boundary.

## Demo Workflow

The demo workflow is:

1. Select approved public source material.
2. Generate a draft pin concept.
3. Draft title, description, alt text, keywords, and board suggestion.
4. Preserve attribution.
5. Flag any uncertainty, unsupported claim, or protected boundary risk.
6. Submit the draft for human/source review.
7. Human/source authority approves, rejects, revises, holds, or requests more evidence.
8. Only approved or properly pre-authorized action may enter reality.

The agent prepares.

The authority decides.

Each Pinterest draft should have a visible status.

Possible statuses:

- draft
- needs review
- needs source
- needs attribution
- needs approval
- approved
- rejected
- hold
- revise
- scheduled by approval
- published by approval

The agent may assign draft status.

The agent may not assign approved status unless approval has been granted by proper authority.

## Approval Boundary

Approval is required before:

- publishing a pin
- scheduling a pin
- modifying a public post
- changing attribution
- altering source language
- using a new source category
- using a new visual asset
- making public claims beyond the approved source
- connecting the demo to external systems

Approval must be explicit or bounded by pre-authorization.

Output is not approval.

## Pre-Authorization Boundary

The demo may allow bounded pre-authorization only if the action class is clearly defined.

Valid pre-authorization may include:

- use only approved public source material
- generate drafts only
- place drafts into a review queue
- apply approved attribution format
- suggest board placement
- never publish without final approval

Pre-authorization is not unlimited permission.

The agent may not expand pre-authorization by inference.

## Publishing Boundary

Publishing is reality-changing action.

The agent may not publish unless:

- the source material is approved
- the draft is approved
- attribution is preserved
- the platform action is authorized
- the action is inside defined scope
- no protected material is exposed
- the stop condition has not been triggered

If any condition is missing, the agent must not publish.

## Scheduling Boundary

Scheduling is also reality-changing action.

The agent may not schedule unless scheduling has been explicitly approved or the scheduling action class has been bounded and pre-authorized.

A scheduled post is a future public action.

Future action still requires authority.

The agent may prepare a proposed schedule.

A proposed schedule is not approval.

## Visual Content Boundary

The agent may suggest visual concepts for public-safe pins.

The agent may not generate or use visuals that:

- expose private material
- imply unauthorized claims
- misrepresent AI Foundations
- erase authorship
- generalize Origin
- imitate private source mechanics
- suggest the demo is the full framework
- use protected imagery without approval

Visual output must remain below source authority.

## Unsupported Claims Boundary

The agent must not create unsupported public claims.

If the source material does not support a claim, the agent must flag it.

The agent may suggest a weaker public-safe phrasing.

The agent may request source confirmation.

The agent may hold the draft.

The agent may not publish unsupported claims.

## Provenance Boundary

The agent must preserve provenance.

The agent may not:

- remove attribution
- replace citation with summary
- treat public material as ownerless
- convert authored claims into generic content
- present agent-generated summaries as original source
- imply that the Pinterest draft is the source framework

Output is not provenance.

Pinterest content is derivative communication, not source record.

## Stop Conditions

The agent must stop when:

- approval is missing
- source material is not approved
- attribution is unclear
- evidence is insufficient
- a claim is unsupported
- protected material appears
- public/private boundary is unclear
- source-position may be blurred
- publishing or scheduling is requested without authority
- the action exceeds demo scope
- the agent cannot determine validity

Stopping protects the demo.

## Agent Action on Behalf of a User

This framework does not prohibit agents from acting on behalf of a user.

In this demo, the agent may act for the user only when authority has been explicitly delegated within a defined scope.

The agent may prepare Pinterest drafts for review.

The agent may not become the user, replace the user’s authority, approve its own action as if the user approved it, or redefine the source boundary.
