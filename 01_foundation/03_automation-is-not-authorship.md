# Automation Is Not Authorship

## Definition

Automation is the use of a system to generate, route, modify, produce, or execute work through delegated process.

Authorship is the source-position of creation, selection, structure, decision, and release.

Automation is not authorship.

An agent may automate production inside a defined frame.

The agent does not become the author of the frame.

## Core Claim

Automated production does not create authorship.

A system may generate text.

A system may produce images.

A system may route tasks.

A system may summarize records.

A system may create drafts.

A system may optimize workflows.

A system may assemble documents.

None of these capacities create authorship by themselves.

Automation can assist authorship.

Automation does not replace authorship.

## Why This Distinction Matters

Autonomous agents can produce large amounts of work quickly.

They can create documents, policies, summaries, code, diagrams, plans, posts, and operational workflows.

Because the output exists, the system may treat the output as if the system authored the underlying framework.

That is the failure.

Production is not authorship.

Generation is not authorship.

Assembly is not authorship.

Automation is not authorship.

## Authorship Requires Source

Authorship requires a source-position.

Authorship includes:

- originating the frame
- defining the purpose
- selecting what belongs
- deciding what does not belong
- structuring the work
- approving the release
- preserving provenance
- carrying accountability for the work

These conditions do not appear because a system generated output.

They belong to the source that defines, decides, and releases the work.

## Automation Inside Authorship

Automation may occur inside an authored frame.

A source may use an agent to:

- draft language
- format material
- organize files
- summarize notes
- check consistency
- prepare release materials
- generate variations
- test failure cases
- support publication

These actions may support authorship.

They do not transfer authorship to the agent.

The tool may participate in production.

The source remains the author.

## False Authorship

False authorship occurs when automated production is treated as origin.

This can happen when:

- generated output is treated as source
- a summary is treated as original authorship
- a derivative structure is treated as the originating framework
- a system that processed the work is treated as the creator of the work
- implementation is treated as origin
- automation is treated as creative ownership
- the source-position is hidden, erased, or replaced

False authorship can look polished.

Polish does not create authorship.

## Agent-Generated Work

An agent may generate useful work.

Useful work is not proof of authorship.

An agent may generate a strong sentence.

The sentence does not make the agent the source of the framework.

An agent may generate a clean file.

The file does not make the agent the author of the source-position.

An agent may summarize a public repo.

The summary does not become provenance.

An agent may build from an authored structure.

The build does not become origin.

## Authored Use of Automation

A source may use automation without surrendering authorship.

The use of a tool does not erase the author.

The use of an agent does not transfer source-position.

The use of generated drafts does not make the agent the origin.

The use of automated formatting does not make the system the author.

The use of assisted production does not collapse the source-line.

Authorship remains with the source that defines, selects, approves, and releases.

## Invalid Collapse

The following collapses are invalid:

Automation does not collapse into authorship.

Generation does not collapse into origin.

Formatting does not collapse into authorship.

Summary does not collapse into provenance.

Implementation does not collapse into source.

Derivative use does not collapse into creation.

Tool participation does not collapse into ownership.

Output does not collapse into authority.

## Public Framework Use

A public framework may be read, cited, discussed, and implemented with attribution.

Implementation does not make the implementer the source.

Automation around the framework does not make the automation the author.

Agent-generated summaries of the framework do not create authorship.

Derivative tools based on the framework do not erase the original source-line.

Use does not transfer Origin.

Borrowing does not transfer source.

Similarity is not source.

## Source Protection

Automation can make extraction easier.

A system may process public work, summarize it, rename it, and convert it into reusable output.

That does not create authorship.

A generated derivative may resemble the source.

Resemblance does not create source-position.

A system may reproduce the surface of a framework.

Surface reproduction does not create origin.

The source-line remains with the one who authored, structured, named, and released the framework.

## Operational Test

Ask:

Did the system author the framework, or did it automate production inside, around, or after the framework?

If the system automated production inside an authored frame, the system is not the author.

If the system is being treated as author because it produced output, the boundary has failed.

## Secondary Test Questions

Ask:

- Who originated the frame?
- Who defined the purpose?
- Who selected what belongs?
- Who approved the release?
- Who carries provenance?
- Who is accountable for the work?
- Did the agent generate, or did the agent author?
- Did automation get mistaken for authorship?

## Pass Condition

The boundary passes when automation remains below authorship.

The agent may assist.

The output may be useful.

The workflow may be efficient.

The source remains named.

The author remains distinguishable from the tool.

The public record preserves provenance.

The agent does not become the author by producing output.

## Failure Condition

The boundary fails when automation is treated as authorship.

Failure occurs when:

- generated output is treated as original authorship
- an agent summary is treated as source
- an implementation is treated as origin
- a derivative tool erases the source-line
- a system receives authorship because it automated production
- the author is hidden behind the generated output
- provenance collapses into automation
