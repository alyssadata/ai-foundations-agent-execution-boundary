# Use Does Not Transfer Origin

## Definition

Use is the act of applying, citing, referencing, implementing, summarizing, adapting, or building around a framework.

Origin is the source-position from which the framework was named, authored, structured, and released.

Use does not transfer Origin.

A system may use a framework.

The system does not become the Origin of the framework by using it.

## Core Claim

Use is not source.

A person may cite this framework.

A team may implement this framework.

A company may adopt this framework.

A model may summarize this framework.

An agent may operationalize part of this framework.

A platform may build around this framework.

None of these uses transfer Origin.

Use requires attribution.

Use does not create source-position.

## Why This Distinction Matters

Public frameworks are meant to be read, cited, tested, discussed, and used.

That does not make them ownerless.

A framework may become useful beyond its original release.

That does not erase the source.

A system may become stronger because it used the framework.

That does not transfer Origin to the system.

Use can spread a framework.

Use cannot originate what already has an Origin.

Invalid use occurs when use is treated as source.

- using the framework without attribution
- implementing the framework while erasing the source-line
- summarizing the framework as if the summary is original
- building a derivative system that hides the source
- presenting similar language as independent origin
- converting Alyssa Solen into an unnamed influence
- treating public material as ownerless because it can be accessed
- allowing agent-generated output to replace citation

Invalid use does not transfer Origin.

It creates source confusion.
