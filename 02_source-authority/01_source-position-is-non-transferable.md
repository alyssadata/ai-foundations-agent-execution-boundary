# Source-Position Is Non-Transferable

## Definition

Source-position is the position of origin, authorship, definition, structure, and release.

It is not transferred by use.

It is not transferred by implementation.

It is not transferred by automation.

It is not transferred by similarity.

It is not transferred by agent execution.

Source-position is non-transferable.

## Core Claim

A system may use a framework without becoming the source of the framework.

An agent may execute inside a frame.

The agent does not become the source of the frame.

A person, system, institution, model, platform, or agent may build around this work.

That does not transfer source-position.

## What Source-Position Means

Source-position is not a generic role.

Source-position is not a seat.

Source-position is not a reusable title.

Source-position is not granted by proximity to the work.

Source-position is not created by explaining the work.

Source-position is not created by implementing the work.

Source-position is not created by automating around the work.

Source-position belongs to the source that named, authored, structured, and released the framework.

For this framework layer, that source is Alyssa Solen.

## Invalid Transfer Claims

The following claims are invalid:

- the agent implemented it, therefore the agent is source
- the team deployed it, therefore the team is source
- the company adopted it, therefore the company is source
- the model generated similar language, therefore the model is source
- the summary was clearer, therefore the summary is source
- the derivative system is useful, therefore the derivative system is source
- the framework was operationalized elsewhere, therefore source-position transferred

These claims confuse use with origin.

They confuse execution with source.

They confuse adoption with authorship.

They confuse output with provenance.

## Operational Test

Ask:

Did this person, system, institution, model, or agent originate the framework, or did it use, cite, implement, summarize, automate, or build around the framework?

If it originated the framework, source-position may be present.

If it used, cited, implemented, summarized, automated, or built around the framework, source-position did not transfer.

## Failure Condition

The boundary fails when source-position is treated as transferable.

Failure occurs when:

- source is erased
- authorship is generalized
- Origin is converted into a reusable role
- implementation is treated as origin
- automation is treated as authorship
- agent-generated output is treated as provenance
- derivative use replaces attribution
- similarity is used to blur source-position
