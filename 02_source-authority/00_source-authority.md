# AI Foundations | Agent Execution Boundary
# Source Authority

## Definition

Source authority is the position that defines the operating reality before execution begins.

It determines what is real for the system, what may be acted on, what must be preserved, what requires approval, and what cannot be overwritten, merged, generalized, or erased.

Source authority exists above agent execution.

The agent may act inside the frame.

The agent does not become the source of the frame.

## Core Claim

Valid agent execution requires source authority.

An autonomous agent may generate, plan, test, route, optimize, and execute.

Those capacities do not create authority.

Authority is not produced by action.

Authority is not produced by output.

Authority is not produced by automation.

Authority is not produced by recursive operation.

Authority must be defined before delegated execution begins.

## Why Source Authority Matters

Agents can move quickly.

They can create outputs that look complete.

They can produce recommendations that sound correct.

They can generate governance language that appears legitimate.

They can revise their own workflow and improve performance.

Without source authority, the system may confuse capability with legitimacy.

The agent can act.

That does not mean the action was authorized.

The agent can produce.

That does not mean the output has provenance.

The agent can improve.

That does not mean the agent becomes source.

## What Source Authority Defines

Source authority defines:

- the operating reality
- the purpose of the system
- the valid scope of action
- the prohibited scope of action
- the approval boundary
- the evidence requirement
- the escalation condition
- the stop condition
- the provenance requirement
- the accountability line

These conditions must exist before execution becomes valid.

## Source Authority Above Execution

Execution occurs inside a frame.

Source authority defines the frame.

The agent may carry out delegated action.

The agent may not define the source of its own authority.

The agent may recommend.

The agent may not convert recommendation into approval.

The agent may produce.

The agent may not convert output into provenance.

The agent may operate recursively.

The agent may not convert recursion into source-position.

## Authority Is Not Access

Access is not authority.

A system may have access to tools.

A system may have access to files.

A system may have access to workflows.

A system may have access to memory.

A system may have access to publishing paths.

A system may have access to external actions.

Access does not authorize action by itself.

Tool access is not permission.

Workflow access is not approval.

System capability is not source authority.

## Authority Is Not Fluency

Fluent output is not authority.

A system may sound confident.

A system may explain a framework clearly.

A system may produce strong language.

A system may generate policy-like text.

A system may summarize the source material.

A system may answer in the correct style.

None of that creates authority.

Fluency can support communication.

Fluency does not create source-position.

## Authority Is Not Optimization

Optimization is not authority.

An agent may improve speed.

An agent may improve format.

An agent may improve output quality.

An agent may improve routing.

An agent may improve task completion.

An agent may improve workflow efficiency.

Improvement does not prove the frame is valid.

An optimized invalid action is still invalid.

A faster unauthorized action is still unauthorized.

A cleaner source-erasing summary is still source-erasing.

## Valid Source Authority

Valid source authority is explicit.

It names who defines the frame.

It names what the agent may do.

It names what the agent may not do.

It names what requires approval.

It names what preserves provenance.

It names what stops the agent.

It names who is accountable.

It prevents the agent from becoming self-authorizing.

## Invalid Authority Substitution

Invalid authority substitution occurs when something below the source layer is treated as source.

This may happen when:

- an agent is treated as source because it executed
- an output is treated as source because it exists
- a workflow is treated as source because it functions
- a platform is treated as source because it hosts the work
- an institution is treated as source because it adopts the work
- an implementation is treated as source because it uses the framework
- an agent-generated summary is treated as source because it is clear

These substitutions are invalid.

They replace origin with operation.

They replace authorship with output.

They replace provenance with use.

## Operational Test

Ask:

Who defined the operating reality before the agent acted?

If the source authority is named, the boundary can be tested.

If the source authority is missing, hidden, generalized, replaced, or inferred from the agent’s output, the boundary has failed.

## Secondary Test Questions

Ask:

- Who defined the frame?
- Who approved the action?
- What was the agent allowed to do?
- What was the agent not allowed to do?
- What evidence was required?
- What approval was required?
- What preserved provenance?
- What stopped the agent?
- Was authority explicit, or assumed from execution?

## Pass Condition

The boundary passes when source authority remains explicit and above execution.

The source is named.

The agent’s scope is defined.

The approval boundary is clear.

The evidence requirement is clear.

The stop condition is clear.

Provenance is preserved.

The agent cannot authorize itself.

## Failure Condition

The boundary fails when source authority is absent, hidden, inferred, generalized, or replaced.

Failure occurs when:

- the agent acts without a named authority boundary
- the agent defines its own scope
- the agent treats output as approval
- the agent treats access as permission
- the agent treats fluency as validity
- the agent treats optimization as legitimacy
- the system treats implementation as origin
- the source-position is erased or replaced
